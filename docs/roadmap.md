# Roadmap posterior al RC

## P0

- Resolver bugs bloqueantes o pérdida de datos si aparecen.
- Confirmar/reemplazar assets y definir términos/canal privado antes de publicación.

## P1

- Validar Preview/Production, Safari/Firefox/Edge y dispositivos físicos.
- Completar round-trip avanzado y exportación en escenas reales.
- Revisar Supabase/rate limit si se habilitan públicamente.

## P2

- Mejorar dataset fotográfico y precisión medible de paredes.
- Evaluar un provider real solo con contrato, privacidad y presupuesto definidos.
- Medir sesión prolongada, memoria y diferencias DOM/Canvas.

## P3

- Ajustes de UX/visuales derivados de feedback real.

## P4

- Funciones futuras solo tras estabilizar 1.0. Cuentas/nube se evaluarán si existe una necesidad comprobada; no se presuponen.

Durante el release candidate no se añaden funciones grandes: se corrigen bugs reproducibles, se mide precisión y rendimiento, y se conserva compatibilidad.
