# Política de seguridad

## Versiones soportadas

| Versión | Soporte |
|---|---|
| 0.9.0-rc.x | correcciones de seguridad y estabilidad durante el RC |
| anteriores | sin soporte formal |

## Reportar una vulnerabilidad

No publiques detalles explotables, secretos, imágenes personales ni tokens en issues públicos. Antes de hacer público el proyecto, el propietario debe definir un canal privado de seguridad. Hasta entonces, contacta al propietario por el canal privado existente del proyecto/equipo; no se inventa aquí un correo.

Incluye versión/commit, entorno, impacto, pasos mínimos, evidencia saneada y mitigación sugerida. No pruebes contra datos o deployments ajenos y no extraigas información de usuarios.

## Alcance

Incluye validación de uploads/proyectos, exposición de variables, API de detección, CSP, IndexedDB, Supabase configurado y dependencias. Errores visuales sin impacto de seguridad pueden usar la plantilla normal de bugs.
